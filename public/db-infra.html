06Data Placement
데이터 유형별 저장 위치 원칙
어떤 데이터를 어디에 저장하는지 명확한 원칙을 정의한다. DB와 NAS의 역할을 분리한다.
데이터 유형 저장 위치 상세
텍스트 · 구조화 데이터 PostgreSQL 사용자, 프로젝트, 클라이언트, 감사 로그 등 모든 구조화 데이터. JSONB 컬럼 활용.
AI 대화 · 비용 추적 PostgreSQL 월별 파티션 ai_conversations 월별 자동 파티셔닝(pg_partman). input/output 토큰, 비용($) 컬럼 추적.
문서 텍스트 + 임베딩 PostgreSQL + pgvector 1536차원 벡터 임베딩. HNSW 인덱스(cosine). RAG 검색용 — 유사도 기반 문서 검색.
이미지 · 영상 · PDF NAS + DB 경로 원본 파일은 NAS에 보관, DB에는 경로(nas_path) + sha256 해시 + 메타데이터만 저장.
DB 백업 파일 NAS gpg AES-256 매일 02:00 pg_dump → gpg 암호화 → NAS 전송. 30일 보관, 매주 복원 테스트 자동 실행.
원칙: 텍스트 · 구조화 데이터는 PostgreSQL에, 바이너리 파일은 NAS에. DB에는 항상 경로 + 해시만 저장한다. 이 원칙으로 DB 용량을 최소화하고 백업을 간결하게 유지한다.

Pattern A
대화형 (Chat)
질문 입력 → DB 이력 조회 → Claude API 호출 → 응답 생성 → messages 테이블 저장
컨텍스트 윈도우에 이전 대화 + DB 데이터를 함께 전송. 토큰 비용과 응답 품질을 함께 추적한다.

RAG (검색 증강 생성)

NAS 문서를 벡터화하여 DB에 저장. 사용자 질문과 유사한 문서를 검색해 Claude에 컨텍스트로 제공한다.

08Security
10가지 보안 체계
4단계 DB 계정, VPN 단일 외부 개방, NAS READ-ONLY 6계층 방어로 데이터를 보호한다.
1
4단계 DB 계정 분리
postgres(DBA) / appuser(API) / readonly_user(직원) / backup_user(백업) — 최소 권한 원칙
2
VPN 단일 외부 개방
WireGuard UDP 51820만 외부 노출. DB/API 포트는 VPN 내부에서만 접근 가능
3
SSL/TLS 암호화 통신
PostgreSQL SSL 필수, API HTTPS 필수. 평문 통신 완전 차단
4
NAS READ-ONLY 6계층 방어
마운트 ro + DB readonly + API 읽기전용 + Nginx 경로제한 + chattr + 감사로그
5
Row Level Security (RLS)
테이블별 행 단위 접근 제어. 사용자는 자신의 데이터만 조회 가능
6
감사 로그 (audit_logs)
모든 데이터 변경 이력을 audit_logs 테이블에 자동 기록. 트리거 기반
7
백업 암호화 (gpg AES-256)
DB 덤프 파일을 gpg로 암호화 후 NAS 전송. 키 분리 보관
8
로그인 시도 추적
login_attempts 테이블로 실패 시도 기록. 5회 실패 시 계정 잠금
9
API Key 관리
api_keys 테이블로 발급 · 만료 · 권한 범위 관리. 해시 저장
10
세션 관리 · 자동 만료
user_sessions 테이블. 24시간 미활동 시 자동 만료. 동시 세션 제한
