Setoworks Jellop+α System Guide

세토웍스 크라우드펀딩 통합 운영 플랫폼의 전체 시스템 구조, 운영 체계, 기술 스택을 정리한 공식 문서입니다.

Last updated: 2026-04-12

1시스템 개요

Jellop+α는 세토웍스가 운영하는 크라우드펀딩 통합 마케팅 플랫폼입니다. 5개 펀딩 플랫폼에서 제품을 자동 수집하고, 글로벌 크리에이터를 발굴하여 데이터 기반의 인플루언서 마케팅을 실행합니다.

5개 핵심 서비스

포털

플랫폼 전체 진입점. 5개 서비스로의 네비게이션 허브 역할.

전략 데크

Jellop+α 90일 로드맵과 투자 전략을 담은 프레젠테이션 (22슬라이드).

크리에이터 관리

65+ 인플루언서의 등급, 콘텐츠, 협업 이력을 관리하는 어드민 패널.

운영 대시보드

530+ 크라우드펀딩 제품과 6개 캠페인을 실시간 분석하는 React 대시보드.

크롤링 모니터

5개 플랫폼 97개 카테고리의 크롤링 작업을 실시간으로 모니터링하고 제어.

3크롤링 시스템

제품 크롤링: 5개 플랫폼

코드플랫폼크롤링 방식비고
KSKickstarterPython 로컬 크롤러프록시 필수

MKMakuakeEdge Function일본 플랫폼
WZWadizEdge Function한국 플랫폼
ZZ기타Edge Function추가 소스

크리에이터 수집

  • YouTube API v3 기반 채널/영상 데이터 수집
  • Edge Function (youtube-collector) 으로 Supabase에서 실행
  • 카테고리별, 국가별 검색 키워드 자동 생성

보호 장치

[KS] Kickstarter = 프록시 필수, 8~15초 딜레이 [429] Rate Limit 감지 즉시 작업 중단 [중복] UNIQUE(platform_id, external_id) + upsert merge [이퀄라이저] 카테고리별 ON/OFF 선택 가능

중복 방지 메커니즘

데이터베이스 레벨에서 UNIQUE(platform_id, external_id) 제약 조건을 사용하며, 크롤링 시 upsert (insert on conflict update) 방식으로 기존 데이터를 병합합니다. 이퀄라이저를 통해 카테고리별로 크롤링을 ON/OFF 할 수 있습니다.

YouTube API 쿼터

일일 쿼터

10,000 units / day
search = 100 units, channels.list = 1 unit, videos.list = 1 unit
초기화: 매일 오후 4시 KST (태평양 시간 자정)

최적화 전략

10채널/회 제한으로 쿼터 소진 방지. 신규 발굴은 09:30 이후 남은 쿼터로 실행.
