System Guide v1.0

Setoworks Jellop+α 운영 매뉴얼

세토웍스 크라우드펀딩 통합 운영 플랫폼의 전체 시스템 구조, 운영 체계, 기술 스택을 정리한 공식 문서입니다. 포털·대시보드·인플루언서 관리·크롤링 모니터·전략 데크의 각 컴포넌트가 어떻게 맞물려 작동하는지 한 페이지에 정리했습니다.

Last updated · 2026-05-07
01

시스템 개요

Jellop+α는 세토웍스가 운영하는 크라우드펀딩 통합 마케팅 플랫폼입니다. 5개 펀딩 플랫폼에서 제품을 자동 수집하고, 글로벌 크리에이터를 발굴하여 데이터 기반의 인플루언서 마케팅을 실행합니다.

📘
정의 · Jellop+α 미국 1위 크라우드펀딩 마케팅사 Jellop을 벤치마크하되, 자체 Alpha 기능(인플루언서 시딩, AI 크롤링, Pure Score)을 결합한 SaaS 모델입니다.

Must Read

멀티 인스턴스 충돌 방지 규칙

이 프로젝트는 동시에 여러 Claude Code 인스턴스가 같은 폴더에서 실행될 수 있습니다. 작업 시작 전 반드시 담당 영역(Owner Lock)을 확인하고, 공유 파일 수정 시 git pull --rebase를 먼저 실행하세요. force push는 절대 금지입니다.

02

크리에이터 관리 체계

인플루언서 데이터는 Pure Score 기반 자동 등급 산정 → 라이프사이클 관리 → 협업 이력 추적의 3단계로 운영됩니다. 13개 카테고리, 8개 국가에 걸친 글로벌 크리에이터 풀을 단일 기준으로 비교 평가합니다.

03

크롤링 시스템

제품 크롤링은 5개 펀딩 플랫폼을 대상으로 하며, 로컬 Python 크롤러와 Supabase Edge Function이 분산되어 동작합니다. 크리에이터 수집은 YouTube API v3 기반의 클라우드 워커가 담당합니다.

제품 크롤링 · 5개 플랫폼

코드플랫폼크롤링 방식비고
KSKickstarterPython 로컬 크롤러프록시 필수
IGIndiegogoPython 로컬 크롤러직접 접근 가능
MKMakuakeEdge Function일본 플랫폼
WZWadizEdge Function한국 플랫폼
ZZ기타Edge Function추가 소스

크리에이터 수집

  • YouTube API v3 기반 채널 / 영상 데이터 수집
  • Edge Function (youtube-collector) 으로 Supabase에서 실행
  • 카테고리별, 국가별 검색 키워드 자동 생성

보호 장치

[KS] Kickstarter = 프록시 필수, 8~15초 딜레이 [429] Rate Limit 감지 즉시 작업 중단 [중복] UNIQUE(platform_id, external_id) + upsert merge [이퀄라이저] 카테고리별 ON / OFF 선택 가능
⚠️
주의 · 프록시 미사용 시 차단 위험 Kickstarter는 직접 접근 시 IP 차단이 발생할 수 있으므로, KS 크롤러 실행 전 프록시 설정과 8–15초 무작위 딜레이가 필수입니다.

중복 방지 메커니즘

데이터베이스 레벨에서 UNIQUE(platform_id, external_id) 제약 조건을 사용하며, 크롤링 시 upsert (insert on conflict update) 방식으로 기존 데이터를 병합합니다. 이퀄라이저를 통해 카테고리별로 크롤링을 ON / OFF 할 수 있습니다.

04

자동화 스케줄

Supabase의 auto_schedules 테이블이 모든 자동화 작업의 단일 진실 원본이며, Edge Function이 cron 트리거로 실행됩니다.

작업주기시간상세
채널 업데이트일일09:00기존 채널 통계 갱신 (10채널 / 회)
신규 발굴일일09:30카테고리별 3채널 × 8개국 = 최대 24채널
C등급 리뷰월간—30일마다 C등급 크리에이터 재평가
아카이브 리뷰분기—90일마다 archived 크리에이터 재검토

YouTube API 쿼터

일일 쿼터
10,000

units / day · search = 100, channels.list = 1, videos.list = 1
초기화: 매일 오후 4시 KST (태평양 시간 자정)

최적화 전략
10채널/회 제한

쿼터 소진 방지를 위해 회당 10채널로 제한. 신규 발굴은 09:30 이후 남은 쿼터로 실행.

🧠
팁 · 쿼터 효율화 search.list는 1회당 100 unit이므로 키워드를 묶어 최소 호출로 다중 결과를 얻고, 후속 channels.list로 상세 통계를 보강하세요.
05

데이터베이스 (Supabase)

Supabase PostgreSQL 기반으로 모든 데이터를 관리하며, RLS는 개발 기간 동안 anon SELECT / INSERT / UPDATE 를 허용합니다.

Supabase URL: https://skcdrvzcwemhjtchfdtt.supabase.co Edge Functions: crawl-worker (제품) | youtube-collector (크리에이터)
